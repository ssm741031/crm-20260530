## 목표
사용자가 **활동(부모) → 세부(자식) 2단 트리** 카테고리를 직접 추가·수정·삭제할 수 있는 화면을 만든다.
(예: "보험설계사 업무" 아래 "장기보험/화재보험", "BNI 활동" 등)

## 왜 이것부터 (추천 근거)
- 카테고리는 할 일·고객이 참조하는 **기준 데이터**라 먼저 있어야 한다.
- 작고 독립적(다른 화면에 의존 안 함)이라 "plan→구현→검증→기록" 한 사이클을 익히기 좋다.

## 리스크 / 주의 (금지사항 체크리스트 연동)
- '기본' 삭제 차단을 빠뜨리면 미선택 할 일이 갈 곳을 잃음 → 반드시 가드
- 2단 초과 트리 생성 방지(자식의 자식 금지) — UI·로직 양쪽 차단
- 삭제는 destructive → **확인창 필수**, 자식 동반 삭제는 명시 경고
- OneDrive Write 충돌 주의: 파일 작성 후 즉시 검증(금지사항 2026-05-30)
- 목 상태는 새로고침 시 초기화됨(정상) — 영속화는 서버 연결 시

## 자기비판 (이 계획의 약점)
- 목 상태를 모듈 메모리에 두면 새로고침 시 사라진다. 토대 단계라 의도된 동작이나, 사용자가 "저장 안 됨"으로 오해할 수 있어 화면에 안내 한 줄 권장.
- 드래그로 부모-자식 재배치까지는 이번 범위에서 제외(추가·수정·삭제까지만). 필요 시 다음 스프린트.
