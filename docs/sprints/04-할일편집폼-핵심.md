# Sprint 04: 할 일 편집폼 (핵심) + 목록 추가/수정/삭제

> 작성일: 2026-05-30 · 상태: 진행중 (2026-05-30 승인됨)
> 관련 계획서 섹션: §2.1(할 일 통합 모델), §5.1(시간 타입), §5.6(편집·태그)
> 담당: 화면·구동로직 = 작업자 / 서버 = 이팀장
> 선행: Sprint 02(토대)·03(카테고리) 완료

## 목표
할 일을 **새로 만들고 / 고치고 / 지우는** 편집폼을 만든다. 카테고리(트리)와 연결하고,
마감형·구간형 시간을 **15분 단위 드롭다운**으로 입력한다.

## 범위 (이번 스프린트)
포함: 제목, 고객(선택), 카테고리(트리 선택), 마감형/구간형 토글, 날짜+15분 시간, 태그, 완료 토글, 추가/수정/삭제
**제외(다음 스프린트로)**: 반복 5종, 알람(최대5), 공유, 첨부 — 폼 자리만 비워둠

## 왜 이 순서
- 할 일은 데이터의 중심(고객·카테고리·캘린더·파이프라인이 모두 연결).
- 캘린더(다음 큰 작업)는 "할 일 데이터가 있어야" 그릴 수 있으므로 편집폼이 먼저.

## 완료 기준 (테스트 가능하게)
- [ ] 목록 화면에서 "새 할 일" 버튼 → 편집폼이 열린다
- [ ] 제목 입력 + 저장 → 목록에 새 할 일이 나타난다
- [ ] 카테고리를 트리(부모>자식)에서 선택할 수 있다 (미선택 시 '기본')
- [ ] **마감형/구간형 토글**: 마감형=종료 날짜·시간만 / 구간형=시작·종료 둘 다
- [ ] 시간은 **15분 단위 드롭다운**(00:00,00:15,…,23:45)으로만 선택된다
- [ ] 태그를 Enter로 추가/삭제할 수 있다
- [ ] 기존 할 일을 클릭 → 폼에 값이 채워지고 수정·저장된다
- [ ] 삭제(확인창) → 목록에서 사라진다
- [ ] 완료 체크 토글이 동작한다 (취소선)
- [ ] 모든 변경은 `api/`를 통해서만 (서버 교체 지점 유지)

## 영향 받는 파일 (예상)
- `frontend/src/api/index.ts` — 할 일 CRUD(create/update/delete/toggleDone) 추가
- `frontend/src/pages/TasksPage.tsx` — 목록 + "새 할 일" 버튼 + 편집폼 열기
- `frontend/src/components/TaskForm.tsx` — 편집폼 (신규)
- `frontend/src/components/TaskForm.css` — 폼 스타일 (신규)
- `frontend/src/utils/time.ts` — 15분 단위 시간 옵션 생성 (신규)

## 리스크 / 주의 (금지사항 연동)
- 시간 입력은 자유 텍스트 금지 → **드롭다운만** (계획서 15분 단위 원칙)
- 구간형에서 시작>종료 같은 모순 입력 방지(간단 검증)
- 목 상태라 새로고침 시 초기화(안내 유지)
- OneDrive Write 후 즉시 검증(금지사항 2026-05-30)
- 반복/알람은 이번에 구현하지 않음 — 폼에 "다음 단계" 표시만, 가짜 동작 금지

## 자기비판 (이 계획의 약점)
- 폼이 커서 한 화면에 다 넣으면 모바일에서 길어질 수 있음 → 섹션 구분으로 완화, 필요 시 다음 스프린트에서 모바일 최적화.
- 고객 선택은 지금 단순 드롭다운(검색 없음). 고객 수 많아지면 검색 필요 — 후순위.
