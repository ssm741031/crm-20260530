## 목표
할 일 편집폼의 **자리표시**였던 반복·알람을 실제 입력 UI로 구현한다.
반복 5종(요일 지정 포함)과 알람 최대 5개(시점+동작 3종)를 폼에서 설정·저장한다.

## 범위
포함:
- 반복 5종: 반복 안 함 / 매일 / 요일 지정 / 매주 / 매월 / 매년 (계획서 §5-B.1)
- 요일 지정 시 월~일 다중 선택 (repeatDays: 0=일~6=토)
- 알람 최대 5개: 시점(정시/15분/30분/1시간/1일 전 + 사용자 지정 분) + 동작 3종(작업완료/연장/알람만)
- 알람 추가/삭제, 5개 초과 방지
- 위 값을 `api`를 통해 저장·수정, 목록 칩에 "반복" 표시(이미 있음) 유지

**제외(명시적으로 다음 스프린트)**:
- 자동 재생성(완료 시 갱신·안내목표·리드타임, §5-B.3) → Sprint 06 (Task 타입 확장 + 완료 트리거 로직 필요. 사용자 결정 2026-05-30)
- 습관 뷰(streak·주간 그리드, §5-B.2) → 별도
- 캘린더 뷰(§5.2) → 별도
- 알람 **실제 발송/푸시**(스케줄러) → 서버·Phase 영역. 이번엔 **설정 저장만**, 가짜 발송 동작 금지(금지사항 연동)
- 공유·첨부 자리표시 유지

## 왜 이 순서
- 반복·알람은 할 일 데이터의 핵심 속성. 캘린더·습관 뷰가 이 데이터를 전제로 그려진다.
- 자동 재생성은 타입 확장+완료 트리거 로직이라 분리해야 폼 작업이 작고 검증 가능.
