## 목표
할 일을 **일·주·월·연 4종 뷰**로 보고, 뷰 전환·날짜 이동·필터를 할 수 있다.
캘린더의 항목을 클릭하면 기존 편집폼(TaskForm)으로 수정·삭제한다. (드래그는 다음 스프린트)

## 범위 (사용자 결정 2026-05-30: 4종 읽기전용 / 드래그 제외)
포함:
- `CalendarPage` + `/calendar` 라우트(자리표시 → 실제)
- 뷰 전환 4종 + 날짜 이동(이전/다음/오늘)
- **월간**: 달력 그리드, ●마감형/▭구간형 칩, 칸 초과 시 "+N" 압축, 빈 칸 더블클릭 → 새 할 일(그 날짜 프리필), 칩 클릭 → 편집
- **주간**: 7일 칸, 칩 표시, 요일 헤더 클릭 → 일간 진입
- **일간**: 운영시간 시간대 그리드, 마감형=마커/구간형=막대(읽기), 칩 클릭 → 편집, 좌우 날짜 이동
- **연간**: 12개월 카드(월별 건수 + 카테고리 색 점), 카드 클릭 → 해당 월간
- 보기 제어(§5.5): 완료 표시/숨김 · 반복 표시/숨김 · 카테고리별 필터(색 토글) — 4종 뷰 공통 적용
- TaskForm 재사용 + `initialDate` 프리필(소형 prop 추가)

**제외(다음/서버)**:
- 드래그 이동·시간 막대 조작(§5.3·§5.7) → Sprint 08 (Pointer Event + 터치 라이브러리)
- §5.4 30분 겹침 "외 N개" 호버 펼침 정밀 동작 → 월간 "+N"만 우선, 호버펼침은 후순위
- 검색(§5.6 제목·고객·태그 통합검색) → 별도
- 종일 타입 → 데이터 모델은 마감/구간 2종만(종일 미도입)

## 자기비판 (이 계획의 약점)
- 일간 "시간대 그리드"는 드래그 없는 읽기 전용이라 막대/마커가 정적 → 다음 스프린트 드래그의 토대로만. 운영시간(8~20) 범위는 우선 고정, 설정화는 후속.
- 반복 항목이 캘린더에 "주기마다 자동 등장"(§5-B.1)하는 전개는 이번 범위 밖(저장된 항목만 표시). 사용자에게 명확히 안내 필요.
- CalendarPage CRUD가 TasksPage와 중복 → 유지보수 부담. 두 곳이 어긋나지 않게 api/ 통로 일원화는 지킴.
