# Sprint 09: 드래그 — 칩 날짜 이동 + 일간 시간 막대 이동

> 관련 계획서 섹션: §5.3(드래그 이동), §5.7(Pointer Event·15분 스냅)
> 담당: 화면·구동로직 = 작업자 / 서버 = 이팀장
> 선행: Sprint 07(캘린더)·08(습관) 완료 (2652995)

## 목표
캘린더에서 **칩을 끌어 다른 날짜로 이동**(월간·주간)하고, **일간에서 막대를 끌어 시각을 변경**(15분 스냅)한다.
모두 **네이티브 Pointer Event**로 구현 — PC 마우스·모바일 터치 단일 코드(§5.7), **새 라이브러리 도입 없음**.

## 설계 방침 (중요)
- HTML5 DnD는 모바일 미지원(§5.7) → **Pointer Event(pointerdown/move/up)** 사용. 외부 의존성 0.
- **클릭 vs 드래그 구분**: 이동 임계값(예: 6px) 넘어야 드래그로 간주, 아니면 기존 클릭(편집 열기) 유지.
- 변경은 전부 `api.updateTask`(목 → 서버 동일) 경유. 화면 상태는 refresh로 갱신.

## 범위 (사용자 요청: 칩 날짜 이동 + 일간 시간 막대 조작)
포함:
- **월간/주간 칩 드래그 → 날짜 이동**
  - 마감형: `endDate` = 드롭한 칸 날짜
  - 구간형: 기간 길이 유지하며 `startDate`·`endDate`를 같은 일수만큼 평행 이동
  - 드롭 대상은 `elementFromPoint`로 날짜 칸(data-date) 판정, 대상 칸 하이라이트
- **일간 막대/마커 세로 드래그 → 시각 이동 (15분 스냅)**
  - 마감형: `endTime` 변경(15분 단위 스냅), 운영시간(07~21) 경계 클램프
  - 구간형: 길이 유지하며 `startTime`·`endTime` 평행 이동
- 드래그 중 시각 피드백(고스트/하이라이트), 드롭 시 저장 + 갱신

**제외(다음/서버)**:
- 일간 **양끝 리사이즈(길이 조절)** → 후속(이번엔 "이동"만). 자기비판 참조
- 목록 내 순서 변경 드래그(§5.3) → 후속
- 캘린더↔다른 화면 간 드래그, 외부 캘린더 → Phase
- 반복 항목을 끌었을 때의 "이 회차만/전체" 분기 → 서버 모델 확정 후(이번엔 저장된 단일 항목만 이동, 안내)

## 자기비판 (이 계획의 약점)
- 일간 "이동"만 하고 "양끝 길이 조절"은 제외 → 계획서 §5.1의 길이 조절은 후속. 우선 이동으로 드래그 토대 검증.
- 미리보기 자동검증은 pointer 이벤트를 dispatchEvent로 시뮬레이션 → 실제 손가락 터치와 미세차이 가능. 핵심 경로(좌표→날짜/시각 변환·저장) 위주로 검증하고, 실제 터치는 사용자 확인 권장.
- elementFromPoint 기반 드롭은 칸에 data-date가 정확히 있어야 함 → 칸마다 data-date 부여로 보강.
