**반복이 켜진 할 일(=습관)** 만 모아 보여주는 화면. 오늘 체크로 streak가 오르내리고,
주간 달성 그리드로 요일별 적용 여부를 본다. 항목 더블클릭 → 기존 편집폼.

## 범위 (드래그 대신 습관 뷰 선택 — 사용자 "계속 진행")
포함:
- 내비 항목 "습관" 추가 + `/habits` 라우트 + `HabitsPage`
- 습관 목록: 제목·카테고리 색·반복종류·🔥streak 표시

- 반복 인스턴스 자동 전개·알림 발송 = 서버
- 드래그 = Sprint 09 후순위(외부 라이브러리 검토 필요)

## 자기비판 (이 계획의 약점)
- 주간 그리드가 "기록 없음" 칸이 많아 당장은 정보가 적다 → 서버에서 일자별 완료이력(check_log) 제공 시 채워짐. 이번엔 구조·오늘 체크·적용판정까지가 목표임을 화면에 명시.
- streak를 토글로 ±1 하는 건 "오늘 1회"를 가정 — 같은 날 여러 번 토글 시 ±1 반복(정합). 자정 경계·중복 체크 방지는 서버 일자 검증으로 보강 예정.
- 습관과 자동재생성(Sprint 06) streak 의미가 살짝 다름(연속 달성 vs 회차) → 같은 필드 공용, 서버 모델 확정 시 정리(결정사항 보류 연동).
