## 4. 내일 IT이팀장과 시작하는 법

### Step 0. 함께 읽기 (30분)
1. **사내CRM_개발계획서_v2.5.md** — 무엇을 만드는지 전체 그림
2. **CLAUDE.md** — 어떻게 일할지 규칙 (특히 0장 작업절차, 1장 백업, 2장 오류방지)
3. 프로토타입 몇 개 실행해보며 "이렇게 보인다" 감 잡기

### Step 1. 선결 과제 확인 (먼저 결정해야 함)
01-프로젝트기초-인증권한.md 하단의 4가지를 IT이팀장과 점검:
- ☐ 고보험 서버 HTTPS 적용 여부
- ☐ 고보험 서버 여유 자원 (CPU/메모리/디스크)
- ☐ 기존 고보험 서비스와 포트·DB 충돌 여부
- ☐ 인증 방식 (세션 / JWT) 택1

### Step 2. 첫 작업 착수 (Sprint 01)
선결 과제가 정리되면, **01-프로젝트기초-인증권한.md의 7단계**를 순서대로:
1. 저장소 초기화 + .gitignore + 구조 잡기
2. PostgreSQL + User 테이블
3. 대표 계정 생성 (※ 비밀번호는 본인이 직접 입력)
4. 로그인 API
5. 프론트 로그인 화면
6. 권한 미들웨어
7. 고보험 서버 배포 + HTTPS 확인

### Step 3. 작업하며 지킬 것
- 매 작업 후 **작업로그 작성** (날짜별, 커밋 해시 포함) → 나중에 "○월 ○일로 복구" 가능
- 오류 나면 **금지사항 체크리스트에 추가**
- 1일 이상 작업은 **sprint plan 먼저 + 승인**
